用一个 multiset 来维护每个块内的数，也可以查询其中小于 $c$ 的最大的数。

NOTE：这个写法超时了。可能需要调整块长。



```cpp
int main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    int n; cin >> n;
    vector<long long> a(n);
    for (int i = 0; i < n; i++)
        cin >> a[i];
    
    int B = (int) sqrt(n);
    int NB = (n + B - 1) / B;

    vector<multiset<long long>> b(NB);
    vector<long long> add(NB);

    for (int i = 0; i < n; i++)
        b[i / B].insert(a[i]);

    int q = n;
    while (q--) {
        int type, l, r, c;
        cin >> type >> l >> r >> c;
        l--;
        int lb = (l + B - 1) / B;
        int rb = r / B;
```

# 关于块数 `NB`

通常我们把块数 `NB` 取为 $\lceil n/B \rceil$。
也就是说我们把最后的长度可能不到 $B$ 的那一段也当作一个块。

```cpp
int NB = (n + B - 1) / B;

auto build = [&](int i) {
    int l = i * B, r = min(n, (i + 1) * B);
    b[i].assign(a.begin() + l, a.begin() + r);
    sort(b[i].begin(), b[i].end());
};

for (int i = 0; i < NB; i++)
    build(i);
```

就这道题来说，如果最后一段长度不到 $B$，我们是不必把它当作一个块的，也就是不必维护那一段数的有序列表。那样做反而麻烦：在 `build()` 里，块的右端点 `r` 需要跟 `n` 取 min。

---

但是 `add` 数组的长度是要定为 $\lceil n / B \rceil$ 的。因为最后一段可能是边块，对于边块我们需要查询它的 `add` 值。
```cpp
else {
```

---

一个可行的办法：
把 `NB` 的值置为 $\lfloor n/ B \rfloor$。
关于块的信息，如果是只有整块才需要的信息，就把对应的数组长度定为 `NB`；
如果是边块也需要的信息，就把对应的数组长度定为 `NB + 1`。

```cpp
int NB = n / B;

vector<vector<long long>> b(NB);
vector<long long> add(NB + 1);

auto build = [&](int i) {
    int l = i * B, r = (i + 1) * B;
    b[i].assign(a.begin() + l, a.begin() + r);
    sort(b[i].begin(), b[i].end());
};

for (int i = 0; i < NB; i++)
    build(i);
```
